Do a lookup using minimal perfect hashing.

The table is stored as a sequence of "salt" values, then a sequence of values that contain packed key/value pairs. The strategy is to hash twice. The first hash retrieves a salt value that makes the second hash unique. The hash function doesn't have to be very good, just good enough that the resulting map is unique.
